# RemotePower v3.4.1 — Release Notes

**Status:** unreleased (dev)

## Overview

**Theme: bind it together.** v3.4.0 grew the fleet outward — health probes,
forecasting, compliance, discovery. v3.4.1 connects those surfaces. Three
cohesion features turn a stack of separate pages into one story, and the
command palette stitches the navigation together.

## Per-device Timeline

A new **Monitoring → Timeline** page gives you a single chronological history
for one host. It merges two streams that previously lived apart:

- **Fleet events** — every event that fired for the device (offline/online,
  config drift, new CVEs, service up/down, SMART failures, brute-force,
  new ports, …), with the event's severity.
- **Command runs** — the commands and scripts executed on the host, with their
  exit status. These are *not* fleet events, so a per-host history never showed
  them before this release.

Pick a device from the selector, filter by category with the chips, and read
top-down. Severity uses the same pill vocabulary as the rest of the app. You can
deep-link straight to a device's timeline from the command palette or the home
health panel.

**API:** `GET /api/devices/<id>/timeline?limit=<n>&kinds=<csv>` — returns
`{device_id, device_name, items: [{ts, kind, severity, title, detail}], kinds,
total}`, newest first.

## Fleet health score

The home dashboard gains a **Fleet health** panel: a single 0–100 score and a
letter grade (good / fair / poor / critical), a severity breakdown, and the
lowest-scoring devices — each a click-through into its timeline.

The score is a weighted rollup of the **same Needs Attention signals** the rest
of the app already computes (offline, patches, CVEs, drift, disk, services,
certs, …). Because it shares that one source of truth, the score can never
disagree with the Needs Attention list. Each device starts at 100 and loses
points per attention item by severity; the fleet score is the mean of the
per-device scores. Only monitored devices count.

**API:** `GET /api/fleet/health` — `{score, grade, devices: [...], counts,
total_devices}`. Also embedded in the `/api/home` bundle, so the dashboard panel
costs no extra request (it reuses the cached attention digest).

## Fleet posture Report + scheduled email

A new **Planning → Reports** page produces one report that binds **patches,
CVEs, the health score, and compliance** into a single artifact:

- **On demand:** download as **JSON** or **CSV**.
- **Scheduled:** email the report on a **cron schedule** to a recipient list
  (or the configured notification recipients). Configuring the schedule is
  admin-only; delivery runs on the heartbeat with a once-per-minute lock so a
  busy fleet never double-sends.

**API:** `GET /api/report/fleet?format=json|csv`,
`GET /api/report/schedule`, `PUT /api/report/schedule`
(`{enabled, cron, recipients}`).

## Command palette, extended

`Ctrl/Cmd-K` (or `/`) now also indexes the new **Timeline**, **Reports**,
**Alerts**, **Compliance**, and **Forecast** pages; offers a
**"<device> — timeline"** jump for every host; a one-keystroke **fleet report
download**; and your **saved scripts**.

## Upgrade notes

No migration required. The new endpoints are additive, the report schedule is
off until you enable it, and the health score is derived from existing data —
nothing new to collect on the agents.
