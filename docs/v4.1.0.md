# RemotePower v4.1.0 — "VisualMatters"

## The story behind this release

By v4.0 RemotePower had grown a *gazillion* functions — every release piled on
more probes, more signals, more telemetry from the agent. The capability was
there. What was missing was a way to **see** it: so much was collected, but
buried, scattered, or simply never surfaced. Power without a view is just noise.

So v4.1.0 deliberately stopped adding new plumbing and turned the focus inward —
to **overview, customization, widgets (especially), design and UX**. The point
of the release isn't another data source; it's making everything already there
*visible, arrangeable, and yours*: a one-screen health matrix, a dashboard you
compose from a catalog of widgets, a sidebar that reads as a clean hierarchy, and
an alert inbox that tells a story instead of a wall of rows. Hence the codename —
**VisualMatters**.

---

This release is about **surfacing what the fleet already knows** and letting you
shape how you see it. The agent has long collected far more than the UI showed;
v4.1.0 binds that telemetry into first-class views, adds a CheckMK-style health
matrix, makes the dashboard something you compose yourself, and quietens alert
storms down to their root cause.

No breaking changes. Most features surface data the agent already reports. After
upgrading, hard-reload the dashboard once so the new front-end loads
(service-worker cache `remotepower-shell-v4.1.0`).

---

## Per-host Checks — a CheckMK-style health matrix

A new **Checks** page (under **Monitoring**) shows every monitored signal on
every host as **OK / WARN / CRIT / UNKNOWN**, with the actual output, sortable
and filterable. It's derived entirely from data RemotePower already collects —
reachability, CPU/memory/swap load, per-mount disk **and inode** usage,
file-descriptor and conntrack-table pressure, failed units, scheduled timers,
config drift, world-exposed ports, pending updates, CVEs, SMART/UPS/temperature,
clock sync, gateway reachability, OOM kills, mail-queue depth, read-only
filesystems, disk-fill ETA, and storage/RAID health.

- **Toggle a check off** to silence it on a host; **Hide muted / ignored** and
  **Hide unmonitored** are on by default so the matrix stays signal.
- **World-exposed-port** checks now respect the mutes you set on the **Exposed**
  page — a muted socket no longer re-appears as a warning here.
- **Custom checks** you define (below) and your **custom monitoring scripts**
  show up as rows alongside everything else.

## Custom checks — assignable to host, tag, group or the whole fleet

Define your own checks from the Checks page:

- **Server-evaluated** (from data already reported): *process running*,
  *port open*, *port closed*.
- **Host-evaluated** (pushed to the agent, run on-host, read-only): *file
  present / absent*, *job freshness* (a stamp file's age), *log error rate*
  (`journalctl -g <pattern>` count over a window, with warn/crit thresholds).

## More monitors — DNS, ICMP, HTTP assertions, DB liveness, tag/group targeting

Active monitors gained DNS resolution (with an expected-address assertion), ICMP
latency + packet-loss thresholds, HTTP status + latency-SLA assertions, and a
**credential-less database-liveness** probe (PostgreSQL / MySQL / Redis — it
speaks just enough of each protocol to confirm the server answers, with no stored
password). Ping / ICMP / TCP monitors can now target a **tag or group** and fan
out to every matching host.

## A dashboard you compose

The Home dashboard is now a **resizable grid of widgets you choose**:

- **65 widgets** spanning inventory, patching/CVE, health/capacity,
  posture/security, hardware, infra state, certificates, bandwidth and
  ops/alerts — added from an alphabetical catalog.
- **Resize** each widget (small / medium / large), **reorder**, **show/hide**,
  **reset** to defaults, **align** to a uniform width, and **share** your layout
  with a teammate via an import/export code. Saved per account.
- New **Upcoming** card (next events from the calendar + scheduler), a **Tickets**
  card (open alerts with one-click ack, plus the last five acknowledged with who
  and when), an actionable **Alerts** widget (ack / resolve / investigate inline),
  and a one-click **quick-ack**.

## Smarter alerts — host grouping + root-cause folding

The alert inbox now **stacks open alerts under a per-host header** (worst
severity + counts, worst first) with group-level **Ack-all / Resolve-all**. When
a host is offline, its downstream symptom alerts (service down, monitors,
metrics, containers, SNMP, health, log, gateway, mount) **fold under the
`device_offline` root cause**, so a storm reads as one incident instead of a
scroll of noise. A classic flat view is one toggle away.

## Security & hardening

- **TLS floor everywhere.** The relay **satellite** (both the agent→satellite
  and satellite→server hops), the **agent**, and the **server's outbound HTTPS**
  now refuse anything below **TLS 1.2** — obsolete TLS 1.0/1.1 can't be
  negotiated even where the platform still offers it.
- Defense-in-depth on the SSH command builder (rejects a host/user beginning with
  `-`, so it can never be smuggled as an ssh option).
- Independently scanned with **wapiti, nikto, nuclei, bandit and OWASP ZAP** —
  clean. See [security.md](security.md).

## Reliability

- The agent's **mail-queue probe backs off for an hour** when the local MTA is
  broken (e.g. a postfix `showq` setuid failure), so it stops re-invoking
  `postqueue` and flooding the journal.
- The dashboard only computes its two heavy widgets (the fleet checks roll-up and
  disk-fill ETA) when you actually display them.
