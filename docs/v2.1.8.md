# v2.1.8 release notes

Released: 2026-05-15.

Hotfix for a v2.1.7 bug where the AI fleet context reported every
device as offline regardless of actual heartbeat state.

## Fixed

### AI said online devices were offline

**Symptom (reported on a 2.1.7 box pointing at Ollama qwen2.5-coder:14b)**:

> What info do you have on tviweb01.tvipper.com?
>
> From your fleet snapshot, here are the details specific to
> tviweb01.tvipper.com:
> * Operating System: Ubuntu 24.04.4
> * **Status: Offline**

…on a device the dashboard was clearly showing as online.

**Root cause.** The fleet-context builder in `ai_context.py` was
reading `d.get('online')` directly from each device dict. But
`online` is a **derived field** — it's computed on-the-fly by
`handle_devices_list` from `last_seen` + `get_online_ttl()`, and is
*not* persisted in `devices.json`. Reading the raw file gave every
device `online=None`, which evaluates falsy, so every device was
labelled offline in the AI's fleet context.

The bug was hidden by two v2.1.7 tests that seeded `online: True`
directly into their device dicts — never failing, but also never
matching production data shape.

**Fix.** `ai_context._is_online(d, now, ttl)` computes status
canonically using the same formula as `handle_devices_list`:

- Agentless devices: `manual_status` (default True)
- Agent devices: `(now - last_seen) < ttl`

`build_fleet_context` and `build_combined_system_prompt` now accept
`now` and `ttl` parameters. `handle_ai_chat` and
`handle_runbook_generate` pass `int(time.time())` and
`get_online_ttl()` so the AI sees exactly the same online/offline
state the dashboard shows.

### Tests

Five new tests in `test_v217.py` covering the regression:

- `test_recent_last_seen_is_online` — recent heartbeat → online
- `test_stale_last_seen_is_offline` — stale heartbeat → offline
- `test_no_last_seen_is_offline` — never-seen device → offline
- `test_agentless_default_online` — agentless with no manual_status → online
- `test_agentless_manual_offline` — agentless with manual_status:false → offline

Two pre-existing tests rewritten to use `last_seen` instead of the
phantom `online` field, so they actually exercise the real code path.

Total: **746 tests, all passing.**

## Upgrading from 2.1.7

Drop-in. No schema changes. If you upgraded to 2.1.7 and noticed
the AI giving you wrong status info, that's this bug — upgrade and
the next AI call will see the right state.

Worth re-running ✨ Generate runbook on any device where the
runbook text mentions offline/stale status — those were generated
under the bug and may include incorrect assertions about device
state.
