# v6.2.3 — "Un1fyMatters"

A release built around **unifying the monitoring surface**: one catalog of
baseline checks you apply by scope, a single "fix this first" CVE ranking,
tickets that carry the full alert and heal themselves, and a page that watches
RemotePower's own internals. Plus a WG Access improvement, a set of fixes, and
a project-wide consolidation + finalize sweep.

## Baseline checks catalog

A shipped catalog of recommended custom checks — Core liveness, Security
posture, Filesystem/OS and Role-tagged groups — applied fleet-wide or by
tag/group in one action (the checks analogue of Service baselines). Applied
checks are ordinary custom checks afterwards: edit or extend them per host.
Checks page → **Baseline catalog**.

## Exposure-weighted CVE ranking

A "fix this first" view ranks every host with open CVEs by one score that
combines *how bad* (critical/high counts, fixable first), *how reachable* (a
world-exposed listening port dominates loopback-only) and *how important* (the
asset's new CMDB `criticality` field). A critical CVE on a world-exposed,
business-critical host lands at the top.

## Tickets carry the full alert — and heal

A ticket opened from an alert now carries every affected host and signal (not
the truncated inbox row), and when the underlying alert clears, the linked
ticket is updated automatically with the recovery.

## Self-observability

The ~33 background maintenance sweeps and the out-of-band scheduler record
their last run, duration and any swallowed error to a ring buffer, surfaced on
a new self-observability view — a silently-failing sweep is now visible. See
[self-monitoring.md](self-monitoring.md).

## Create tunnel — optional listen port

The **Create tunnel** dialog (Settings-independent, on the WG Access page) gains
an optional **Listen port** field:

- Leave it blank and the hub picks the next free UDP port, exactly as before.
- Set it to pin the tunnel to a specific port — handy when you've already opened
  one on the firewall or want a predictable endpoint.
- The port is validated (1–65535) and rejected if another tunnel already uses it.
- Like the interface name and address pool, it's fixed once the tunnel exists, so
  the field is shown read-only when editing.

Full guide: [wg-access.md](wg-access.md).

## Fixes

- **ACME certificates load on navigation.** Opening the TLS page used to leave
  the ACME panel on "Loading…" until you pressed Refresh. Both the certificate
  expiry and ACME panels now populate automatically when you open the page.
- **Package Snapshots gives clear feedback for an empty capture.** Creating a
  snapshot when no host has reported a full package inventory yet now says so
  explicitly, instead of a success toast that was easy to miss.
- **No more duplicated dampening controls.** Settings → General used to repeat
  the "Metric/SNMP alert dampening" fields that also live on
  Settings → Alert parameters — and both wrote the same config values, so
  saving one page could quietly reset the other. Alert parameters is now the
  single home for them.

## Under the hood — consolidation sweep

A whole-project pass to reduce duplication and dead weight, all behaviour-
preserving:

- The request-body validation pre-check that appears in ~240 API handlers was
  collapsed into a single shared helper — about 480 fewer lines, identical
  behaviour.
- Verified-dead frontend code was removed, and several duplicated JavaScript
  helpers (relative-time, HTML-escaping, byte formatting) were unified.
- Repeated backend validation helpers (port-range checks, secret scrubbing)
  were consolidated, and native browser confirmation prompts were migrated to
  the in-app dialog that doesn't get throttled during fleet-wide actions.
- The **Storage & hardware** doc is now an umbrella that links the focused
  disk-health, GPU, thermal and power guides instead of restating them.

## Finalize sweep

A pre-release whole-project pass:

- **Four new guides** — [ipam.md](ipam.md), [oncall.md](oncall.md),
  [sso.md](sso.md) and [secret-scan.md](secret-scan.md) — plus ~30 in-app
  "Documentation" pointers on pages and Settings sections that lacked one.
- **~225 new translation entries** across all 6 languages (Alert parameters,
  on-call, tickets, alerts, webhook log, logs, AI/CVE/compliance strings).
- **LDAP honours the group → role map** (`sso_group_roles`, keys = group DNs,
  case-insensitive) — LDAP users are no longer admin-or-viewer only.
- Wiring fixes: Fleet Query's `os` column, the status-page monitor picker, a
  drawer-set poll interval on Linux agents, container names in the activity
  feed, and "Scan now" for the secrets scan (Windows agents + a UI button).
- Perf (`_load_ro`) on the NOC board, network metrics and scoped alert views;
  header-band and icon-gap consistency fixes; a fully refreshed demo dataset.
