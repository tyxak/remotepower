# v6.2.3 — "Un1fyMatters"

A consolidation and tidy-up pass. One WG Access improvement and two UI fixes on
the surface; underneath, a project-wide sweep that collapsed repeated code
idioms, removed a duplicated setting and dead code, and de-duplicated the docs.
Nothing changes in how the product behaves beyond the fixes below.

## Create tunnel — optional listen port

The **Create tunnel** dialog (Settings-independent, on the WG Access page) gains
an optional **Listen port** field:

- Leave it blank and the hub picks the next free UDP port, exactly as before.
- Set it to pin the tunnel to a specific port — handy when you've already opened
  one on the firewall or want a predictable endpoint.
- The port is validated (1–65535) and rejected if another tunnel already uses it.
- Like the interface name and address pool, it's fixed once the tunnel exists, so
  the field is shown read-only when editing.

Full guide: [wg-access.md](wg-access.md).

## Fixes

- **ACME certificates load on navigation.** Opening the TLS page used to leave
  the ACME panel on "Loading…" until you pressed Refresh. Both the certificate
  expiry and ACME panels now populate automatically when you open the page.
- **Package Snapshots gives clear feedback for an empty capture.** Creating a
  snapshot when no host has reported a full package inventory yet now says so
  explicitly, instead of a success toast that was easy to miss.
- **No more duplicated dampening controls.** Settings → General used to repeat
  the "Metric/SNMP alert dampening" fields that also live on
  Settings → Alert parameters — and both wrote the same config values, so
  saving one page could quietly reset the other. Alert parameters is now the
  single home for them.

## Under the hood — consolidation sweep

A whole-project pass to reduce duplication and dead weight, all behaviour-
preserving:

- The request-body validation pre-check that appears in ~240 API handlers was
  collapsed into a single shared helper — about 480 fewer lines, identical
  behaviour.
- Verified-dead frontend code was removed, and several duplicated JavaScript
  helpers (relative-time, HTML-escaping, byte formatting) were unified.
- Repeated backend validation helpers (port-range checks, secret scrubbing)
  were consolidated, and native browser confirmation prompts were migrated to
  the in-app dialog that doesn't get throttled during fleet-wide actions.
- The **Storage & hardware** doc is now an umbrella that links the focused
  disk-health, GPU, thermal and power guides instead of restating them.
