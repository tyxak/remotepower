# v2.4.4 release notes

Released: 2026-05-17.

A bugfix-and-polish release: it makes the 2.4.3 mailbox monitor
actually work, moves its configuration to Settings, restyles the
dashboard view as a tile, and restores favicon.ico.

## Fixed

### Mailbox monitor never received its paths (the 2.4.3 bug)

v2.4.3 shipped the mailbox monitor, but it never counted anything:
the dashboard sat on "no report yet" indefinitely.

Cause: the heartbeat handler caches a few device fields into a
`saved_dev` snapshot inside its file lock, then builds the agent
response from that snapshot. The mailbox path list was read from
`saved_dev` — but it was never copied *into* `saved_dev`. So the
agent received `mailbox_paths: []` on every heartbeat, was never
told which directory to count, and never reported a number. (The
agent log made this visible: it logs every config push, and there
was no "mailbox_paths" line — only services and watched_files.)

Fixed: `saved_dev` now carries `mailbox_paths`, so the heartbeat
response actually contains the configured paths. A new test
asserts a heartbeat response includes them — the gap that let
this ship was that 2.4.3's tests covered the config endpoint and
the agent counter but not the heartbeat that connects them.

### favicon.ico restored

Browsers automatically request `/favicon.ico` whether or not a
page references it. The project shipped only `favicon.png`, so
`/favicon.ico` returned 404. A real `favicon.ico` (16/32/48 px,
generated from the existing icon) is now included in the document
root and referenced from the page.

## Changed

### Mailbox monitor configuration moved to Settings

Configuration was on each device's detail modal — easy to miss.
It now has its own **Settings → Mailbox monitor** tab: pick a
device, enter its mailbox paths, tick the dashboard option, Save.

### Dashboard view is now a tile

The mailbox count was a separate full-width card. It is now a
tile in the Home dashboard's top row — the same style and size as
"Devices online", "Pending updates", "Drift events" and "CVE
findings". The tile shows total unread mail across the promoted
devices and appears only when at least one device is promoted.

## Tests

`tests/test_v244.py` (7 tests): the heartbeat response includes
configured `mailbox_paths`; the `saved_dev` fix is present;
favicon.ico exists, is a valid ICO, and is referenced; the
Settings tab exists; the dashboard tile replaced the standalone
widget; the device-detail section was removed. `test_v243.py`
asset tests updated for the relocations.

**Total: 1003 tests, all passing.**

## Upgrading from 2.4.3

Drop-in. If you configured a mailbox path in 2.4.3 it is still
stored on the device and will start working once 2.4.4's server
is deployed — the path was always saved correctly; only the
heartbeat push was broken. Re-deploy is enough; no reconfigure
needed. Make sure favicon.ico lands in the document root with the
rest of `server/html`.

## Honest caveats

1. **The end-to-end agent path is still only logic-tested.** The
   heartbeat now provably includes the paths (new test), and the
   agent's counter and config-pickup are unit-tested — but I have
   still not watched a real agent count a real Maildir and a
   number land on the dashboard. With the heartbeat bug fixed
   this *should* now work end to end; confirm it on the one host
   before relying on it. Two heartbeats after saving, a count
   should appear.

2. **Counts refresh every ~5 minutes.** Unchanged from 2.4.3 — the
   agent reports mailbox counts every ~5 polls, not every poll.

3. **Browser-untested.** The Settings tab, the dashboard tile, and
   the favicon are covered by asset/unit tests, not a real browser
   load.

4. **No threshold alerting.** Still just a count; no webhook when
   it crosses a number. A natural follow-up.
