# RemotePower v4.4.1 — "DocumentationMatters"

A documentation-and-triage release on the heels of the security-themed v4.4.0.
It resolves the open **CodeQL code-scanning alerts**, runs a documentation-
coverage pass across every doc surface, and applies two cheap, no-behaviour
hardenings so the scanner agrees with the code. **No breaking changes and no
runtime behaviour changes**; every install keeps working exactly as before.

After upgrading, hard-reload the dashboard once so the new front-end loads
(service-worker cache `remotepower-shell-v4.4.1`).

## CodeQL alert triage — all 13 are false positives

Every open CodeQL alert (1 Critical, 12 High) was read against the actual code
path and the feature that governs it. None is an exploitable vulnerability. The
full write-up, with the feature/function that covers each, is in
[security-review-4.4.1.md](security-review-4.4.1.md). In summary:

- **XSS / DOM-text-as-HTML (7 alerts).** Every `innerHTML` assignment flagged in
  `app.js` / `app-compliance.js` interpolates only values that pass through the
  project's `escHtml()` (entity-encode) or `escAttr()` (neutralise every
  HTML-breaking character) sanitizers — which CodeQL's default model doesn't
  recognise. The strict no-inline CSP is an independent second barrier.
- **SSRF (1 Critical).** `fetch('/api' + path)` is the same-origin front-end API
  client; the target is an origin-pinned relative path, never a user-supplied
  host. Not request forgery.
- **Weak hashing (2 alerts).** One points at the audit-log chain, which is
  **HMAC-SHA256** — strong. The other is a non-security dedup key already marked
  `usedforsecurity=False`.
- **Clear-text logging / storage (3 alerts).** The admin-gated diagnostics
  download (by design, `no-store`), a non-sensitive `stderr` diagnostic line,
  and the application's own atomic `chmod 0600` data-file write — none is a
  secret-disclosure path.

## Cheap hardening (no behaviour change)

- The two **MD5 cache-key fingerprints** in the fleet-checks fast path now pass
  `usedforsecurity=False` — the signal CodeQL's weak-hash query honours. They are
  change-detection fingerprints, never credential digests; cache keys recompute
  transparently. The remaining 11 alerts are documented false positives suitable
  for dismissal on the code-scanning tab.

## Documentation coverage pass

A sweep across all five documentation surfaces — the in-app Documentation page,
`README.md`, `docs/Manual.html`, the per-version `docs/vX.Y.Z.md` notes, and the
did-you-know tips — confirmed every shipped feature and function is documented
and that the headline counts are accurate (65 webhook events, the 66-widget
dashboard catalog, 18 MCP tools = 14 read + 4 guarded write). The security-review
index and the durable [security.md](security.md) pointers were repointed to the
current set.
