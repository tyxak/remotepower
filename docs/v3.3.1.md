# RemotePower v3.3.1 — Release Notes

**Released:** 2026-05-28

## Overview

Correctness + polish release on top of v3.3.0. No breaking changes and
no schema changes. The headline is a rework of OFFLINE detection that
ends the device flapping seen in production logs; the rest is a sweep of
live-instance bugs and UI consistency fixes found while getting the
build ready for public production.

Because the service-worker cache version is bumped, installed PWAs pick
up the UI fixes below automatically on their next load — no manual cache
clear needed.

## Highlights

### OFFLINE detection no longer flaps

The per-request sweep that decides whether a device is offline fired on
a **single sample** crossing the TTL. Any transient — a heartbeat one
second late, a config read landing just before an in-flight heartbeat's
rename, or the lost-update race the `_LockedUpdate` comment describes —
produced an `OFFLINE` → `ONLINE` pair in the same second, spamming the
log and any webhook/email subscribers.

Three layers now sit in front of the decision (`_offline_thresholds`):

- **Grace** — a fixed jitter cushion (`OFFLINE_GRACE_S`) folded into the
  threshold, so a 1–2s late beat never even becomes a candidate.
- **Per-device threshold** — the cutoff is
  `max(global ttl, poll_interval × OFFLINE_MISSED_POLLS) + grace`, so a
  30s poller and a 600s poller no longer share one fixed cutoff, and a
  device that legitimately polls slower than the global TTL isn't
  perpetually flagged offline.
- **Debounce** — the first sweep past the threshold only *arms* a
  candidate (`offline_pending`); `OFFLINE` fires only if a later sweep,
  at least one poll interval on, still sees the device silent. A live
  device beats in between and clears the candidate. Recovery (`ONLINE`)
  stays immediate.

The bar is now **5 missed polls** (`OFFLINE_MISSED_POLLS = 5`) — 300s at
the default 60s poll, matching `DEFAULT_ONLINE_TTL`. `offline_pending` is
managed everywhere `offline_notified` is (monitored-off handlers,
monitor-alerts-clear, config hiding) and is purged on device delete,
which previously leaked `offline_notified` too.

### Agent patch status no longer false-warns

`get_patch_info()` ran `pacman -Sy` and `pacman -Qu` under one
try/except and distinguished failures with `'pacman -Qu' in str(e.cmd)`
— but `str(['pacman','-Qu'])` is `"['pacman', '-Qu']"`, so that
substring never matched. Every `pacman -Qu` exit-1 (the normal "no
upgrades available" path) was misreported as `pacman sync failed
(rc=1); patch status unknown`. The two commands now sit in separate
try blocks, so a clean, fully-patched Arch/CachyOS host reports
correctly.

### Home dashboard counts match the detail pages

`/api/home` aggregated CVE and drift totals by iterating
`cve_findings.json` / `drift_state.json` directly. Those side files keep
records after a device is deleted (no cascade), so the Home tile
over-counted relative to the CVE/Drift pages, which iterate
`devices.json`. Both aggregators now iterate live device ids and look up
the side-file state — matching the detail handlers.

### Settings reflect real runtime defaults

`handle_config_get()` only echoed keys present in `config.json`, so on a
server that never touched the v3.3.0 flags the Settings toggles rendered
"off" even when the runtime default was "on" (notably
`webhook_block_local`). The handler now `setdefault()`s the v3.3.0 flags
and their runtime defaults, so the UI shows the state the runtime
actually uses. No backend semantics change.

### Fleet events honour `?event=`

`handle_fleet_events` ignored the `?event=` query parameter the UI
timeline filter relies on. It now filters by event name (multiple
`?event=` values OR together; an unknown name returns an empty list
rather than silently falling through to "all").

## UI polish

- **Action buttons aligned across pages.** Edit/Delete/Revoke siblings
  mixed 14px SVG icons with raw glyphs (`×` / `✗`) and one-off padding
  classes, so paired buttons were different sizes. They now share a
  class and icon style. The ACME **force-renew** button — which was
  rendering with **no icon at all** — gets its refresh icon.
- **PWA standalone clipping fixed.** In an installed PWA the docked
  ~220px sidebar isn't subtracted from the device-table column
  breakpoints, so a narrow window squeezed `table-layout: fixed` until
  the Status pill collapsed to a lone `…`; the nav scrollbar also clipped
  the "MCP Confirmations" count badge. Both are fixed, scoped to
  `@media (display-mode: standalone)` so the browser UI is untouched.
- **Maintenance windows show the hostname.** A device-scoped window's
  Target column rendered the opaque device id; the list endpoint now
  resolves it to the device name.
- **"Did you know?" tips on the About page** surface lesser-known
  features (credential vault, web terminal, MCP integration, SNMP
  polling, IaC export, and more), with a button to cycle through them.

## Upgrading

Standard agent self-update applies (hash-driven, as of v3.3.0). No
config migration. Operators who lowered `online_ttl` below the default
will now get the 5-missed-poll floor per device via the new threshold.
