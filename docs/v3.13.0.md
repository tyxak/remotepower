# RemotePower v3.13.0 — Bind it together (round four), polish & hardening

The RemotePower agent reports a rich heartbeat about every Linux host — far more
than the UI has historically shown. Over several releases we've been steadily
"binding it together": making sure every signal the agent collects, and every
value the server computes, has a home where an operator can actually see it.

v3.13.0 is the fourth such sweep. It surfaces a batch of collected‑but‑hidden
host signals, caps every panel so it scrolls instead of growing without bound,
and adds a round of performance and security hardening. There are **no breaking
changes** and **no new dependencies** — everything here renders data that was
already being reported and stored.

After upgrading, hard‑reload the dashboard once so the new front‑end loads
(service‑worker cache `remotepower-shell-v3.13.0`).

---

## Newly surfaced host signals

Each of these was already collected by the agent and sanitised/served by the
API — v3.13.0 adds the missing front‑end binding.

### Access — recent logins (device drawer)
The drawer's **System info** card now lists **who logged in and from which
distinct source IPs**, aggregated per user. This is the same `auth.recent_logins`
data the *new login source* alert fires off; until now the underlying
who/from‑where table had no UI. It's the highest‑value security signal on a host,
so it finally gets a table.

### Scheduled jobs / timers
A **failed‑first** table of every systemd timer on the host — the timer unit,
what service it activates, and whether it's currently failed. Mirrors the
treatment failed units already get, and gives the *scheduled‑job failure* lens a
per‑device home.

### Pools / arrays (per‑host storage & RAID health)
This host's own **ZFS / mdadm / btrfs** pools and arrays — state, capacity, and
scrub status — now appear in the drawer. Previously this lived only on the
fleet‑wide **Storage** page, so investigating one host meant leaving it.

### Listening ports — bind address & scope
The drawer **Ports** card gains an **Address** column and a **world / LAN /
local** scope badge for each socket (the same classification the fleet
**Exposure** page uses). You can now see that `:22` is bound `0.0.0.0` (world)
right where you're inspecting the host, rather than having to cross‑reference the
Exposure page.

### Firewall ruleset summary
The **Firewall** card shows the active backend, total active rule count, and a
short fingerprint — the drift baseline the *host firewall changed* alert compares
against — above the per‑backend table.

### Brute‑force lockouts & pressure pills
- Active brute‑force sources (units + source IPs) now show as a **badge on the
  device card**; clicking opens the host's logs.
- The drawer adds at‑a‑glance **Disk** and **Swap** percentage pills alongside
  the existing uptime/load/boot pills.

## Every box now fits (max ~15 rows, then scroll)

Panels that grow with data used to stretch the page arbitrarily. v3.13.0 caps
them so they scroll internally instead:

- **Device‑drawer cards** are capped and scroll per‑card (Pools, Ports, Mounts,
  Containers, SMART, the new timers/logins tables, …).
- **Large page tables** — notably **Compliance** (which can list 100+ controls
  per framework) — scroll inside a fixed‑height, sticky‑header wrapper.
- **Two latent clip bugs are fixed**: the host‑config inspection dump and the
  per‑device patch‑command history previously had a `max-height` with no
  `overflow`, so long content was silently cut off with no scrollbar.

## Performance

- **Immutable static‑asset caching.** Every app asset is version‑busted in its
  URL (`?v=3.13.0`, bumped each release), so it's now served with
  `Cache‑Control: public, max-age=31536000, immutable`. This removes the
  per‑load conditional‑revalidation (304) round‑trips that the bare
  ETag/Last‑Modified default incurred for `app.js`, the CSS, and the feature
  modules.
- **Deferred scripts.** The front‑end scripts now load with `defer`, so parsing
  no longer blocks on them (they were already `DOMContentLoaded`‑gated, so
  ordering and behaviour are unchanged).
- **File‑cached fleet risk.** `_compute_fleet_risk()` is now wrapped in the same
  10‑second file‑backed cache pattern the Needs‑Attention digest uses. Because
  CGI spawns a fresh process per request, this is the only way to share the
  full‑fleet risk computation between `/api/home` (via fleet health) and
  `/api/risk`. The cache is busted whenever devices, CMDB, CVE, software‑policy
  or hardware data changes.

> **Note:** API JSON responses remain deliberately **un‑gzipped** — that's an
> intentional BREACH/CRIME defence on token‑bearing responses, not an
> oversight. Only static assets (already compressed) and now their cache
> lifetime are tuned.

## Security hardening

- **Sandboxed SCAP reports.** Full OpenSCAP/usg HTML reports are supplied
  verbatim by the (device‑token‑authenticated) agent and served back inline.
  They are now emitted under a **self‑contained sandboxed CSP**
  (`default-src 'none'; … sandbox;`) plus `X-Frame-Options: DENY`, so a
  compromised agent can't land stored XSS in an operator's session even if the
  upstream/global CSP is ever loosened. Reports still render (they're static
  tables with inline CSS); only scripts, forms and same‑origin access are
  neutralised.
- **OIDC id_token claim checks.** On top of the existing confidential‑client
  back‑channel trust (TLS + `client_secret` + SSRF‑guarded fetch + state/nonce),
  the callback now rejects an **expired** token (with 120 s clock skew), an
  **issuer** that doesn't match the configured `oidc_issuer`, and an
  **audience** that isn't this client — blocking replay of a leaked token and
  cross‑relying‑party token confusion.
- **Syslog audit‑forward DNS‑rebinding fix.** The syslog SIEM forwarder now
  resolves its target **once** to a literal IP, classifies that IP, and connects
  to the literal — closing the rebinding window between the SSRF pre‑flight check
  and the raw‑socket send (the HTTP forwarder already did connect‑time peer
  revalidation).

## Smaller fixes & polish

- **Numeric tag filter highlight.** A purely numeric device tag now highlights
  correctly when selected (the action dispatcher coerced numeric args to
  `Number`, breaking the strict string comparison used for the active‑pill
  state).
- **HTML escaping.** `escHtml` now also escapes single quotes, matching the
  project's other escape helpers (defence‑in‑depth for single‑quoted
  attributes).
- **Update banner.** The "release notes" link is now scheme‑validated
  (http/https only) before it's rendered.

---

## Upgrade

Standard upgrade — no migration, no config changes:

```bash
# on the server
sudo ./deploy-server.sh
```

Then hard‑reload the dashboard once. Agents need no changes for the UI features;
they already report all of this data. (As always, agents self‑update against the
server's canonical binary hash.)
