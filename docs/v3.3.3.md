# RemotePower v3.3.3 — Release Notes

**Status:** unreleased (dev)

## Overview

A focused, additive release: the Alerts inbox gains a per-alert **AI
Investigate** button. No backend schema changes, no breaking changes.

## AI Investigate on alerts

The Alerts inbox (MAIN → Alerts) already let you acknowledge and
resolve alerts. v3.3.3 adds a third action to every open row, sitting
alongside Ack and Resolve:

- **Investigate** — opens the AI modal pre-loaded with the alert's
  severity, event type, affected device, timestamp, and raw message
  (plus any structured payload the alert carried). The model returns:
  1. one sentence on what the alert means,
  2. the most likely cause, and
  3. 2–4 concrete next steps or commands (`systemctl`, `journalctl`,
     `dig`, `ss`, `df`, …) to verify and resolve it.

It is backed by a new `investigate_alert` system prompt, editable under
**Settings → AI** like every other AI action prompt. As with the rest
of the AI surfaces, the request content is sent to the configured AI
provider — the modal says so, and the data is redacted per your AI
privacy settings before it leaves the server.

### How it differs from the webhook "Explain" button

The Webhook log's **Explain** button (`explain_alert`) rewrites a raw
alert into a single paragraph an on-call engineer can parse in five
seconds. **Investigate** (`investigate_alert`) is the deeper,
action-oriented sibling: it does not just paraphrase — it proposes a
likely cause and the commands to act on it.

## Notes

- The button only renders on open / acknowledged rows; resolved rows
  still just show "resolved by …".
- Because of the cache-first service worker, the button reaches
  installed PWAs once the cache token changes — which this version bump
  (`?v=` + `CACHE_NAME` → v3.3.3) does. Hard-reload once after upgrading.
