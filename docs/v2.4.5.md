# v2.4.5 release notes

Released: 2026-05-17.

A small features release: an on-demand package scan.

## Added

### "Scan packages now" — force a package report on next heartbeat

The agent normally submits its full package inventory (used for
CVE scanning) and the patch / upgradable count only every few
hundred heartbeats — roughly every few hours at the default 60s
interval. That's fine for routine monitoring but frustrating when
you've just patched a host and want RemotePower to reflect it.

The device action menu now has **"Scan packages now"**. It sets a
one-shot flag on the device; the device's next heartbeat response
carries the request, and the agent then sends a fresh package
list and patch count on the heartbeat after that.

The flag is cleared the instant it's handed to the agent — it
fires exactly once. If the agent somehow misses it, click again.

**Timing:** this is not instant. The agent has to receive the
request on one heartbeat and report on the next, so a fresh
inventory lands within roughly one to two minutes at the default
interval — not immediately.

## Tests

`tests/test_v245.py` (7 tests): the endpoint sets the flag, rejects
an unknown device and an invalid id; the heartbeat response
carries `force_package_scan` exactly once and then the flag is
cleared on disk; the second heartbeat does not carry it; the
one-shot pop is present in the source; the agent honours the flag
and the frontend button exists.

**Total: 1010 tests, all passing.**

## Upgrading from 2.4.4

Drop-in. The agent gains the flag-handling behaviour; deploy the
updated agent so hosts can act on the request. An older agent
simply ignores the flag (the button still appears to work — the
request just won't be acted on until the agent is updated).

## Honest caveats

1. **Not instant — one to two minutes.** Covered above. The button
   queues a request; it does not reach out and run a scan
   synchronously. The toast message says as much.

2. **The agent must be on 2.4.5** to act on the request. On an
   older agent the flag is pushed, ignored, and cleared — the
   scan just won't happen. Keep agent and server versions
   aligned.

3. **Browser-untested**, as usual — the menu item and the handler
   are covered by asset/unit tests, not a real browser load.
