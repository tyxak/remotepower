# RemotePower v4.4.0 — "FortifyMatters"

A hardening-and-binding release. The headline is security — a full server +
agent audit, an independent pentest pass, and fixes ranging from a critical
privilege-escalation closure down to defence-in-depth — alongside the usual
bind-it-together pass that surfaces more of what the agent already collects,
plus read-path performance and reliability work. **No breaking changes**; every
install keeps working exactly as before.

After upgrading, hard-reload the dashboard once so the new front-end loads
(service-worker cache `remotepower-shell-v4.4.0`).

## Security — audited, fixed, and independently pentested

This release had every server handler and all three agents (Linux, Windows,
macOS) read end-to-end for injection, SSRF, path traversal, authorization and
secret-handling issues. The fixes:

- **Admin-gate hardening (critical).** The `require_admin` check blocked the
  built-in `viewer` and `mcp` roles by name — but a *custom operator role*
  (assignable to a login account) was neither, so it slipped through every
  admin-only endpoint (user / role / API-key / config management and the
  agent-update signing key). The gate now resolves the role and checks its
  actual `admin` flag, so only the built-in admin role passes. Built-in
  `viewer`/`mcp` behaviour is unchanged; custom roles are now correctly
  confined to their granted permissions and scope.
- **Scope-guarded diagnostic routes.** The mitigation status / AI-analysis
  endpoints (which return captured command output) sit outside the
  `/api/devices/` prefix, so the central role-scope guard didn't cover them — a
  scoped operator could read another scope's diagnostic capture. They now run
  the same per-device scope check as the rest of the device API.
- **Shell-quoting on queued commands.** The drift "fetch file content" action
  and the ACME issue/renew/revoke commands interpolated a path into a shell
  string with hand-rolled quoting. Both now use `shlex.quote`, so an awkward
  watched-file path or agent-reported home directory can't break out of the
  intended command.
- **RouterOS SSRF guard.** The RouterOS REST integration fetched its
  device-configured host with no anti-rebinding check. It now runs the same
  pre-flight that the rest of the egress paths use — loopback and
  link-local/cloud-metadata addresses are refused, while the RFC1918 LAN
  address a real router lives on is allowed.
- **Metrics endpoint can't be knocked over.** `/api/metrics` (Prometheus
  exposition) now degrades to a minimal valid payload plus a
  `remotepower_scrape_error` gauge if a single malformed store record would
  otherwise have thrown — one bad record no longer 500s the whole scrape and
  breaks fleet monitoring.
- **Agent: no predictable-tempfile clobber.** The on-host lynis posture audit
  wrote its report to a fixed `/tmp` path; a local unprivileged user could
  pre-plant a symlink there and have the root agent clobber an arbitrary file.
  It now uses a private, unpredictable temp file and cleans up after itself.
- **Windows & macOS agents enforce HTTPS + a TLS 1.2 floor.** The Linux agent
  already refused plain-HTTP server URLs and pinned a TLS 1.2 minimum; the
  Windows and macOS agents now do the same, so the device token and command
  output can never travel in cleartext and an obsolete-TLS downgrade is
  refused.

**Independently pentested.** This release was scanned with **wapiti, nikto,
nuclei, bandit and OWASP ZAP** and passed clean — no exploitable findings, only
informational items. See [security.md](security.md).

## Bind it together — more agent signals, surfaced

- **Login history with timestamps.** The agent already reported recent logins
  and their source IPs; it now also reports *when* each login happened, and the
  device drawer's **Access — recent logins** table gains a **Last seen** column.
- **Accurate clock verdict.** The drawer's Clock pill now reflects the server's
  threshold-aware skew verdict — a host that is NTP-synced but drifting beyond
  the configured offset threshold reads **skewed** (matching the `clock_skew`
  alert) instead of a plain "synced".

## Faster where it counts

- **Single-row heartbeat reads.** Two remaining hot-path lookups — the
  watched-files list and the host-config enforce check, both run on *every*
  heartbeat — were doing a full-fleet store read after the heartbeat write
  invalidated the cache. They now read a single row, so on the SQLite /
  PostgreSQL backends they're O(1) instead of O(fleet). Identical data; flat-JSON
  backend unchanged.
- **The fleet-checks cache actually caches now.** The 15-second Checks/rollup
  cache was being invalidated by every heartbeat's write to the devices store,
  so on any fleet larger than a handful of hosts it effectively never hit and
  the full per-host checks loop re-ran on every poll. It now honours its TTL;
  the change-fingerprint still busts it instantly when you edit a check, script
  or config (the things you change and expect reflected).
- **Agent memoizes its OS string.** `/etc/os-release` is read once per process
  instead of on every 60-second heartbeat.

## Polish

- **More boxes capped.** A few remaining variable-length lists (the fleet AI
  anomaly results, the saved-links grid, the AI-prompt customization list) now
  cap their height and scroll internally like every other panel.
- **Typography normalized.** A handful of off-scale font sizes left over from
  the CSP inline-style migration were folded back onto the canonical scale.
- **Installer & docs.** The server installer now ships and wires the optional
  SCGI worker unit, copies the full static asset tree, and ends with a clear
  TLS-setup reminder; `docs/install.md` documents the password utility, the
  high-performance worker, and a step-by-step Let's Encrypt path.
