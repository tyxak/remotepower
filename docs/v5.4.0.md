# RemotePower v5.4.0 — "RackMatters"

A lightweight **time-tracking + billing** layer, built on one shared time-entry
ledger: log billable (debtable) or internal hours where the work happens — on a
ticket or a weekly timesheet — then turn those hours, plus recurring fees, into
per-customer **invoices**. Customers pay for the time you spent and the things
you've stamped in. No breaking changes.

After upgrading, hard-reload the dashboard once so the new front-end loads
(service-worker cache `remotepower-shell-v5.4.0`). See the full
[time-tracking & billing guide](time-billing.md).

## One unified time ledger

Every logged hour is a single entry — `hours` (in 0.25-hour steps, 0.25 = 15 min),
a date, who, a billable flag, an optional customer (site) / device / ticket link,
and a note or internal category. "Add hours" on a ticket and the weekly timesheet
are two views of the same records, so nothing is counted twice and everything
rolls up to billing.

- **Hours on tickets.** A **Log hours** button on every ticket. Type the hours
  (0.25 steps), mark them **billable (debtable)** or **internal**. A billable
  entry attaches to the ticket's customer — the site (derived from the ticket's
  device, or picked explicitly). `device`/`tag` are attribution labels; the
  **site is always the payer**.
- **Weekly timesheet.** A personal **Timesheet** page (Planning → Timesheet, and
  linked from **My Account**). A week view you can page back and forth through,
  with per-day and weekly totals (billable vs internal). Stamp in ad-hoc time —
  meetings, education, internal projects, travel, admin — not just ticket work.
  Hours you logged on tickets appear here automatically.

## Billing (Admin → Billing)

Three tabs, gated to admins and the new finance role:

- **Worksheet.** Pick a customer (site) and a month → billable hours grouped by
  rate × rate, plus the customer's active recurring fees, with subtotal / VAT /
  total. Export to CSV, or **Generate invoice**.
- **Invoices.** Issued invoices with a **draft → sent → paid** lifecycle (and
  **void**). Issuing an invoice **snapshots and locks** the hours it bills, so an
  invoice can never silently diverge from its source data — voiding frees those
  hours to be re-billed. View / print to PDF, export CSV.
- **Rates & Fees.** A named **rate card** (e.g. Standard / After-hours / Project),
  a global default rate, currency and default VAT, an invoice-number prefix, and
  **per-customer** overrides: default rate, VAT %, billing contact + address, and
  recurring **license / operation / service** fees.

## Money model

One install **currency** with a **per-customer VAT %** (a global default,
overridable per site). Rates resolve in order: the named rate-card rate → the
customer's default rate → the global default. Amounts are **frozen onto the
invoice's line items at issue**, so a later rate change can't move an issued
invoice.

## New "finance" role

A read-only **finance** role can view and export the billing surfaces
(worksheet, invoices, rates) without admin rights. Issuing/voiding invoices and
editing rate/fee config stay admin-only. Everyone logs their own hours regardless
of role. Assign it on the **Users** page (Edit role).

## Export — XLS / CSV / PDF / API

- **CSV** on the time-entry ledger, the worksheet and each invoice (opens in
  Excel/Sheets; formula-injection-safe).
- **JSON API** on every list: `/api/time-entries`, `/api/timesheet`,
  `/api/billing/worksheet`, `/api/invoices`.
- **PDF** by printing the styled invoice from the browser (no server dependency).

## API surface

`GET/POST /api/time-entries`, `PATCH/DELETE /api/time-entries/{id}`,
`GET/POST /api/tickets/{id}/hours`, `GET /api/timesheet?week=YYYY-Www`,
`GET/POST /api/billing/config`, `GET /api/billing/worksheet?site=&month=`,
`GET/POST /api/invoices`, `GET /api/invoices/{id}`, `PATCH /api/invoices/{id}`.
All list/worksheet/invoice GETs accept `?format=csv`.
