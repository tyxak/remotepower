# v2.4.13 release notes

Released: 2026-05-18.

A documentation and repository-housekeeping release. There are no
server or agent behaviour changes beyond the version bump — if
2.4.12 is running cleanly, this upgrade is purely cosmetic.

## Changed

### `docs/features.md` brought current

The feature inventory was audited against the 2.4.x series. The
lines-of-code figures in the intro were stale (the server is now
~17,600 lines of Python, the agent ~2,100) and have been
corrected. Seven internal links pointed at `docs/<file>.md` from
inside `docs/` itself — they resolved to `docs/docs/<file>.md` and
were broken; all now use repo-relative paths.

### `CHANGES.md` and `CHANGELOG.md` merged

The repository carried two parallel changelogs that had drifted
apart — each held release entries the other was missing. They are
now a single `CHANGELOG.md` containing the union of both, newest
first. `CHANGES.md` has been removed.

### `docs/Manual.html` refreshed

The manual was still describing version 2.2.6. It has been brought
up to date with the current feature set and bumped to 2.4.13. The
byte-identical duplicate copy at the repository root was removed;
`docs/Manual.html` is now the single source.

### Per-release notes pruned

The `docs/` directory accumulated ~45 `vX.Y.Z.md` files. These
duplicated content now consolidated in `CHANGELOG.md`. Only the
three most recent are kept (v2.4.11–v2.4.13); `CHANGELOG.md`
remains the complete history.

### Tags cleaned up

Old and malformed release tags (missing or doubled `v` prefixes,
pre-2.x tags) were removed. Only the last three releases are
tagged going forward.

### `README.md` rewritten

The README now leads with the project logo, followed by the
dashboard screenshot.

## Tests

No code changed, so the test suite is unchanged at 1064 tests.

## Upgrading from 2.4.12

Drop-in. Pull and redeploy:

```bash
cd /path/to/remotepower
git pull origin main
sudo bash deploy-server.sh
```

No API, agent, or data-file changes.
