# v6.3.0 — "Fl0wMatters"

A thirteen-wave UX improvement program plus fleet-visibility and ops fixes:
**undo instead of "are you sure?"**, optimistic alert actions, a notification
center, configuration history with rollback, drafts that survive, shift-click
range selection, laptop battery health, an agent version-skew filter,
remediation runbooks from failing checks — and fixes for the everyday
friction operators hit most.

## Undo instead of confirm

Deleting a low-risk, operator-authored object — a contact, a link, a
command-library snippet, a saved device view — no longer interrupts with a
confirmation dialog. The row disappears immediately and a toast offers
**Undo** for a few seconds. Under the hood the delete is *deferred*: the API
call is only sent when the toast expires, and Undo cancels it outright.
Nothing ever leaves the server until the window closes, so an interrupted
delete always fails toward "still there", never silent loss.

Destructive fan-out operations (Docker prune, device removal, VM rollback,
alert purges) deliberately keep their explicit — sometimes typed —
confirmations. Undo is for the deletes you do ten times a day.

## Optimistic alert actions

- Pressing `a` in the keyboard-driven alert inbox acknowledges the selected
  alert **instantly** — the row flips before the server answers — and the
  success toast carries **Undo**, wired to the real
  `POST /api/alerts/<id>/unack`.
- **Resolve** flips the row optimistically and reverts with an error toast if
  the server declines. (It keeps its confirm dialog: there is no un-resolve.)

## Filtered tables say so

Every table managed by the shared table controller now shows
**"N of M shown — filter active"** with a one-click **Clear filter** chip
whenever the filter hides rows. Table filters persist across sessions, which
used to make a table look mysteriously short days later with no visible cause.

## Retry buttons on failed loads

About a dozen "Failed to load — refresh to retry" dead ends (Checks, CVE
views, CMDB credentials, Thermal / Power / Disk health, Firewall, Rollouts,
GPUs, Drift, Proxmox backups, Compliance baseline, Tuning) now render an
inline **Retry** button that re-invokes the loader — no full-page refresh.

## Unsaved-changes guard on Settings

Editing any setting marks the page dirty. Navigating to another page — or
closing the tab — asks before discarding, and a successful save clears the
flag. (The page's own filter/search boxes don't count as edits.)

## Tickets: bulk actions

All four ticket tables gain row checkboxes and a bulk bar with **Resolve
selected** and **Assign to me** — the multi-select pattern's first outing
beyond the Alerts inbox. Bulk operations reuse the same per-ticket API calls
as the single-row flows.

## Wave 2 — quick wins

- **Ctrl/Cmd-Enter** submits the active modal (its primary button); Esc
  already cancels.
- **No more double submits** — any button whose action is still in flight
  dims and ignores clicks until it settles, automatically, app-wide.
- **Click-to-copy** on CVE ids and command-library commands (`data-copy` —
  renderers can opt any monospace value in with one attribute).
- **Rows-per-page (15/50/100) and CSV export** on every paged table — the
  choice persists per table on your user record; the export honors the
  current filter and sort and includes all pages.
- **Recently-viewed devices** at the top of the command palette (last 8,
  per browser).
- **"A new version of RemotePower is available — Reload"** toast when the
  service worker installs an update while an old page is still running.
- **Skeleton loading** on the remaining bare "Loading…" modal bodies and
  cards.

## Wave 3

- **Per-alert deep links** — the link icon on any alert row copies
  `#alerts/<id>`; opening it lands on the inbox with that alert scrolled into
  view and highlighted (the status filter widens to "all" once if the alert
  has since resolved). Made for pasting into tickets and chat.
- **Notification center** — the topbar bell replays the last 30 toasts of the
  session with timestamps; an unread dot appears when something fired while
  the popover was closed. Session-only by design — alerts and the audit log
  remain the durable records.
- **Alert parameters: visible overrides** — any of the ~120 threshold fields
  that differs from its default gets an accent edge and a small reset button
  (its tooltip names the default); the **Changed only** checkbox collapses
  the page to just your overrides.

## Wave 4

- **Column show/hide** — the columns icon in any table pager opens a checkbox
  menu; hidden columns persist per table on your user record, next to
  filter/sort/rows-per-page. At least one column always stays visible.
- **Dashboard layout undo** — widget toggle/move/resize/align and the full
  reset all register with the topbar undo arrows (reset's toast carries Undo
  too).
- **Page-aware help** — `?` now ends with a "This page" section: the current
  page's guide and its own keyboard map.

## Wave 5

- **Configuration history + rollback** — Settings → Advanced keeps the last
  10 saved configuration states (timestamp, who, which keys changed).
  Restore swaps the live config for a snapshot; the replaced state becomes a
  revision itself, so a restore can always be undone. Listings never expose
  the stored config bodies; restore is admin-only and audited.
- **Pinned devices** — star a host in its drawer; pinned hosts lead the
  command palette and the pins live on your user record (cross-browser).

## Waves 6–8 — drafts and duplication

- **Draft autosave** in the script editor and the KB editor — every keystroke
  snapshots to your browser; reopening with a differing draft offers Restore,
  and a successful save clears it.
- **Duplicate** for saved scripts and custom checks — clone-as-prefill,
  detached from the source.
- **Cross-tab sync** — alert ack/resolve/mute in one tab refreshes the badge
  and inbox in every other open tab immediately; theme and accent changes
  follow across tabs too.
- **Purge confirmations state the real counts** ("Clear 37 resolved alerts?"),
  not "every".

## Wave 9 — the idea batch

Undo history dropdown (right-click the topbar arrow), a "While you were away"
digest after 30+ minutes in another tab, command-queue drained toast,
quiet-hours moon in the topbar, drawer prev/next (`[` `]`), sticky first
column on scrollable tables, reset-view and copy-rows-as-JSON, palette scope
prefixes (`>` actions, `#` devices), human-unit hints on alert parameters,
S/M/L interface scale, and pull-to-refresh on touch devices.

## Wave 10 — informative motion

All behind `prefers-reduced-motion`: a fleet-pulse miniature in the topbar,
count-up stat tiles with delta arrows, a real progress ring on CVE scans,
row-change flash on refreshed tables, a critical-alert pulse, a 6-week alert
heat strip above the inbox, health-bar grade ticks, and calm page/row
entrances.

## Waves 11–12 — selection flow and a quieter bell

- **Shift-click range selection** on device checkboxes (respects filter and
  sort, both densities); **Escape clears the selection**; the select-all
  checkbox reflects partial selections; absolute-time tooltips on "3h ago"
  values. Also fixes the devices table replaying its entrance animation on
  every selection.
- **Form-validation toasts stay out of the notification center** — "Subject
  required" is feedback for the field you're looking at, not an unread
  notification.

## Wave 13 — fleet visibility

- **Laptop battery health** — the Linux agent reports charge, status, cycle
  count and current-vs-design wear for `BAT*` power supplies; shown in the
  device drawer. Servers and VMs without a battery are unaffected.
- **Agent version-skew chip** — when any agent runs an older version than the
  newest in the fleet, the Devices page shows an "N agents outdated" chip
  that filters to exactly those hosts.
- **Remediate from a failing check** — every non-OK row on the Checks page
  gains a wrench that generates a host-scoped remediation runbook via the AI
  advisor (uses your configured provider).

## macOS agent self-update

The mac agent's `update` command now performs a real self-update — HTTPS
no-redirect download, constant-time sha256 verification, atomic install, and
a re-exec into the new file only after the result is reported to the server.
Previously macOS was the one platform whose agents had to be re-pushed by
hand (the command honestly returned "not implemented"). Served by
`/api/agent/mac/{version,download,signature}`; audit-mode agents refuse.

## Ops fixes

- Deploy and guided self-update now **restart the sidecar daemons**
  (scheduler, push relay, webterm, scanner) and refresh the webterm/scanner
  file copies — previously only the app server restarted, so shipped sidecar
  fixes stayed dormant until a manual restart. Rendered per-host unit files
  are never overwritten.

## Toast actions (for integrators)

`toast(msg, type, opts)` accepts an optional third argument:
`{action: {label, fn}, duration, onTimeout, transient}`. Action toasts stay
up 6 s and settle exactly once — either the action fires or `onTimeout` does,
never both. `transient: true` keeps a toast out of the notification center.
