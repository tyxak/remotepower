# v5.6.0 — "ProvisionMatters"

The IaC / automation + alert-tuning release. Everything new is **opt-in and
default-off** — no breaking changes.

## Provisioning (Admin → Provisioning)

An opt-in folder-tree **catalog of infrastructure blueprints** — Terraform,
cloud-init, Ansible and iPXE templates. Enable it under **Settings → Advanced →
Optional features**.

- **Render** any blueprint: fill in its declared variables and copy/download the
  result. Two macros are always available — `${rp_server_url}` and
  `${rp_agent_install}` (the real `curl … /install | sudo sh` one-liner), so a
  cloud-init blueprint can bake the agent and a provisioned box auto-enrols.
- **Run Terraform** server-side — **Plan / Apply / Destroy** — when the separate
  **`iac_execute_enabled`** gate is on (Settings → Optional features) and
  `terraform` is installed on the server. Mirrors the Ansible runner:
  - persistent per-blueprint workdir (`<data-dir>/iac_runs/<id>`) so state
    survives → Destroy and re-apply work;
  - a per-blueprint run lock (no two ops race one blueprint);
  - **secret** variables passed as **environment only** (`TF_VAR_<name>` + the
    bare name, for cloud creds) — never written to disk or the command line;
    non-secret variables become an auto-loaded `tfvars.json`;
  - admin-only and audited; deleting a blueprint that still has **live state** is
    refused until you Destroy it.
- cloud-init / iPXE stay render-only (a booting machine consumes them — there's
  nothing to execute on the server).
- The former standalone **Ansible** page is folded in here as an **Ansible
  playbooks** card.

Render is pure string substitution; Terraform blueprints are returned verbatim on
Render (Terraform owns the `${…}` syntax — values are supplied natively as
`var.<name>` at Run time).

## Alert tuning (Monitoring → Tuning)

A new page that reads the fleet-event **timeline** and surfaces the **top-10
noisiest alerts** (host + event) and **top-10 sources**, each with a **Silence**
button.

- The per-row **Ack** button on the **Alerts** page and the **dashboard** is now
  an **X "Mute"** button. Muting silences one **exact alert from one host** — no
  inbox row, no webhook, no needs-attention — and clears any currently-open
  matching alerts. History (`fleet_events`) keeps recording, so the Tuning page
  always sees what's being silenced and you can lift a mute anytime.
- Mutes are **permanent** until removed under Tuning. Muting is admin-only and
  audited.

## Timesheet watchers

Lets specific **non-finance** users view another user's **timesheet** — read-only,
hours only, never rates. A watch grant targets a single **user** or a whole
**team** (`ui_prefs.team`, resolved live). Admin/finance still see everyone.

- Managed under **Users → Timesheet watchers** (omnisearch pickers).
- A **"Watch for"** omnisearch on the **Timesheet** page switches whose week you
  view; the same gate guards the CSV export (no side-channel).

## Settings

- The opt-in modules — **Tickets, Billing, Provisioning, File manager** — are
  consolidated into a single **"Optional features"** section (they were nested
  under the Tickets heading).
- Fixed the Settings **Save** button hanging on "Saving…" forever when a request
  was dropped — it now resets and surfaces the error.
