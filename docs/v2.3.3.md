# v2.3.3 release notes

Released: 2026-05-17.

A bugfix release.

## Fixed

### Virtualization page was undiscoverable

The Virtualization nav entry (added in 2.3.0) shipped with
`display:none` and was only revealed by a JS call once Proxmox was
enabled in Settings. That created a dead-end: you configure Proxmox
under Settings -> Proxmox, but you couldn't find the feature in the
first place because its nav entry stayed hidden until it was
already configured.

Fixed: the Virtualization nav entry is now always visible. The page
itself already handles the not-configured state — it shows a
"configure it under Settings -> Proxmox" message when Proxmox isn't
set up — so there's no reason to hide the entry. `refreshProxmoxNav()`
is kept as a no-op so existing call sites don't break.

## Tests

Full regression: **940 tests, all passing.** No new tests — this is
a one-line visibility fix to existing, already-tested functionality.

## Upgrading from 2.3.2

Drop-in. One HTML attribute and one JS function changed.

## Known issue — NOT fixed in this release

A separate report of a **broken mobile render** (the page showing
almost nothing — no header, no sidebar, just status dots) is **not
addressed here.** That symptom indicates a fundamental load failure
— most likely a JavaScript error on page load or a stylesheet
problem — and it cannot be responsibly diagnosed without the
browser console error (the red message + file:line shown in the
browser's developer console). Shipping a guessed fix onto a broken
UI risks making it worse. Once the console error is available the
real cause can be found and fixed properly.
