# v2.2.7 release notes

Released: 2026-05-17.

A mobile hotfix. The mobile navigation drawer was unusable — this
fixes it.

## Fixed

### Mobile sidebar drawer was a panel of unlabelled mystery icons

On phones the navigation drawer opened as a wide (240px) panel
showing four bare icons, no text labels, with the icons pushed
halfway down an otherwise empty panel.

Root cause: two CSS media-query blocks were fighting. An older
`max-width: 768px` block styled the sidebar as a 56px **icon-rail**
(labels hidden, buttons centred, 72px top padding). A newer
`max-width: 720px` block restyled it as a 240px slide-in **drawer**.
Below 720px *both* applied — the drawer got its 240px width from one
block while the icon-rail block kept the labels hidden and the
padding oversized.

Fix: the 768px icon-rail block is removed entirely. The ≤720px
drawer is now the single mobile layout, and it explicitly restores
everything the rail had stripped — nav labels and section headers
visible, buttons left-aligned, sane padding. Screens in the
720–768px range now use the normal desktop layout, which is fine at
that width.

Also folded in: the mobile nav overlay/drawer z-index values were
moved onto the v2.2.6 tier scale (800 / 810).

## Tests

`tests/test_v227.py` (6 tests): icon-rail block removed, drawer
restores labels, nav left-aligned, padding reset, braces balanced,
content margin reset.

**Total: 893 tests, all passing.**

## Upgrading from 2.2.6

Drop-in — a single CSS file changed. No server, agent, or data
changes. Redeploy and hard-refresh on mobile (or bump the asset
cache) to pick up the new stylesheet.

## Honest caveats

1. **Browser-untested**, as always — but this is a small, contained
   CSS change to a single layout, and the failure that prompted it
   is well understood. Lower risk than most releases.
2. **720–768px now gets the desktop layout.** A tablet held in
   portrait at ~768px wide will see the full desktop sidebar rather
   than a drawer. That's usable, but if you specifically want a
   drawer at tablet widths too, that's a follow-up — say so.
