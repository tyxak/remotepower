# v2.2.3 release notes

Released: 2026-05-15.

A small hotfix to the Home dashboard activity panel. The 2.2.2
activity feed showed everything in the webhook delivery log —
including operator-triggered SMTP / webhook tests, which on a
freshly-configured server can fill the panel with rows like:

> *test (email) 1 recipient(s): smtp_host is empty*
> *test (email) 1 recipient(s): SMTPNotSupportedError: SMTP AUTH …*
> *test (email) 1 recipient(s): SSLCertVerificationError …*

Real fleet events (device offline, drift detected, CVE found, etc.)
got crowded off the first-8 window. Discord and ntfy deliveries
were already in the log — they ride along with their fleet event
names like `device_offline` — but were invisible behind the test
noise.

## Fixed

**Activity panel now filters to canonical fleet events.** The JS
keeps a `FLEET_EVENTS` allowlist that mirrors the server's
`WEBHOOK_EVENTS` tuple — `device_offline`, `device_online`,
`monitor_down/up`, `service_down/up`, `cve_found`, `patch_alert`,
`log_alert`, `container_stopped/restarting/stale`,
`metric_warning/critical/recovered`, `command_queued/executed`,
`drift_detected`.

The `test` event (used by `_log_email('test', …)` for SMTP test
deliveries, and by `fire_webhook('test', …)` for webhook test
deliveries) is **not** in the allowlist. Tests stay in the
underlying webhook log for the Settings → Webhook log view
(unchanged behaviour) — they just don't clutter the dashboard.

The filter runs **before** the `slice(0, 8)` so real events can't
be crowded off the visible window by a burst of test noise.

The empty-state copy is updated to make the filter explicit:

> *No recent fleet events*
> *Fleet events (device offline, drift detected, CVE found, etc.)
> show up here as they're delivered. Manually-triggered SMTP /
> webhook tests are filtered out.*

## Contract test

`test_v223.TestActivityFilter` verifies:

- The JS `FLEET_EVENTS` set is **exactly equal** to the server's
  `WEBHOOK_EVENT_NAMES` tuple. If a future commit adds a new fleet
  event to the server without updating the JS, the test fails —
  otherwise that new event would silently disappear from the
  dashboard.
- `'test'` is excluded from both the JS allowlist and the server
  canonical list.
- `filter()` is applied before `slice()` in the activity renderer
  (source-order check).
