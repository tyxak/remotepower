# v5.1.0 — "FidelityMatters"

Status: unreleased (test).

A security-signal and localisation release on top of v5.0.1 "TemperMatters".
No breaking changes.

## fail2ban bans are now a first-class event

When a host's fail2ban jail bans a **new** IP, RemotePower fires a first-class
`fail2ban_ban` webhook/alert event. Previously fail2ban bans were recorded only
in the audit log and never surfaced as a signal.

- The event lands in the **Alerts** inbox, the **dashboard activity feed**, and
  any configured **webhook / SIEM** destination, routed through the normal
  notification matrix (kind: `fail2ban`).
- The payload carries the `jail`, a representative `first_ip`, the full list of
  newly-banned `ips` (capped per heartbeat), and a `new_count`.
- Bans are **edge-triggered** against the previous heartbeat snapshot, so only
  genuinely new bans fire; the first snapshot after enrolment is seeded silently
  rather than firing the whole existing ban list.
- Repeat bans on the same host **coalesce** into a single open alert (count
  bumped) so a brute-force flood does not drown the inbox — while each ban is
  still delivered to webhook/SIEM destinations.

`WEBHOOK_EVENTS` is now **82** (this release also adds `av_infected`, below).

## Active malware/rootkit detections now alert

A ClamAV or rkhunter scan that reports an **active infection** now fires the new
`av_infected` event. Previously this was a posture card only, so a real detection
never reached the Alerts inbox or a webhook. It is high-severity, edge-triggered
on the rising infected-count (re-reports of a known infection don't re-fire), and
has no recover event — a malware finding stays actionable until an operator clears
it; repeat reports on a host coalesce into one open alert.

## Arabic right-to-left layout

The Arabic (`ar`) locale already rendered correct right-to-left **text**, but the
layout never mirrored. `styles.css` now carries `[dir="rtl"]` layout overrides so
the sidebar, navigation, cards, tables and drawers flip to a true RTL layout.

## More localized UI strings

Another batch of previously hardcoded-English renderer labels across the
Firewall, Reputation/DMARC, AI Insights, Alerts and Checks pages are now
translated in all five supported languages (en, zh, hi, es, ar).

## App catalog — add your own apps

The App Catalog (one-click Docker Compose deploys) now lets an admin add **custom**
apps alongside the curated set: give it a name and a compose YAML and it joins the
catalog, ready to deploy to any host. Deploys still ride the existing audited,
permission-gated compose path; custom entries can be removed again from the same
page.

## Finalize sweep

This release also had a whole-project finalize sweep — a structured audit across
the entire codebase plus a live authenticated pentest and the local SAST suite:

- **Security.** Fixed a high-severity webhook dead-letter-queue leak (the admin
  listing echoed a secret-bearing destination URL — now redacted to host) and a
  medium-severity SSRF gap (the AI provider re-validates the peer IP at connect).
  Full details in [security-review-5.1.0.md](security-review-5.1.0.md).
- **Performance.** The Needs-Attention, fleet risk/health and nav-counts caches now
  hit on the SQLite/PostgreSQL backend (they were recomputing the full-fleet scan
  on every poll in production).
- **Polish.** Three more variable-length tables now scroll instead of growing;
  three Arabic RTL accent rails mirrored; a Settings toggle for the
  `allow_internal_monitors` SSRF opt-in; the cron / app-catalog / file-manager
  chrome is now fully translated in all five languages.

## Correctness, performance & contributed fixes

- **Device-write race fixed — data loss on the SQL backend.** ~24 device-write API
  handlers did an unlocked read-modify-write of the device store; on the SQLite/
  PostgreSQL backend a slow admin edit could delete a device row (and its auth
  token) for a device that enrolled mid-edit, because the full-set save reconciles
  against a stale snapshot. Every handler now does its read-modify-write under
  `_LockedUpdate(DEVICES_FILE)`. Reported by @tbouquet (#8).
- **The ~50k-line backend is no longer recompiled on every request.** The CGI entry
  point is now a thin `api_cgi.py` shim that runs the backend from cached bytecode
  instead of letting fcgiwrap recompile the main script each time (~0.9s → ~0.15s
  per request). Contributed by @tbouquet (#7).
- **`_sanitize_ip` rejects trailing garbage** (#3), **RAG embedding cache is
  invalidated when the embedding model/provider changes** (#6),
  **`disk_watchdog_pct=0` is honoured in the self-test** (#4), and
  **`PUT /api/drift-policies` accepts a bare-list body again** (#5) — all
  contributed by @tbouquet.
