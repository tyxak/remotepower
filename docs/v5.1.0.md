# v5.1.0 — "VigilMatters"

Status: unreleased (test).

A security-signal and localisation release on top of v5.0.1 "TemperMatters".
No breaking changes.

## fail2ban bans are now a first-class event

When a host's fail2ban jail bans a **new** IP, RemotePower fires a first-class
`fail2ban_ban` webhook/alert event. Previously fail2ban bans were recorded only
in the audit log and never surfaced as a signal.

- The event lands in the **Alerts** inbox, the **dashboard activity feed**, and
  any configured **webhook / SIEM** destination, routed through the normal
  notification matrix (kind: `fail2ban`).
- The payload carries the `jail`, a representative `first_ip`, the full list of
  newly-banned `ips` (capped per heartbeat), and a `new_count`.
- Bans are **edge-triggered** against the previous heartbeat snapshot, so only
  genuinely new bans fire; the first snapshot after enrolment is seeded silently
  rather than firing the whole existing ban list.
- Repeat bans on the same host **coalesce** into a single open alert (count
  bumped) so a brute-force flood does not drown the inbox — while each ban is
  still delivered to webhook/SIEM destinations.

`WEBHOOK_EVENTS` is now **81**.

## Arabic right-to-left layout

The Arabic (`ar`) locale already rendered correct right-to-left **text**, but the
layout never mirrored. `styles.css` now carries `[dir="rtl"]` layout overrides so
the sidebar, navigation, cards, tables and drawers flip to a true RTL layout.

## More localized UI strings

Another batch of previously hardcoded-English renderer labels across the
Firewall, Reputation/DMARC, AI Insights, Alerts and Checks pages are now
translated in all five supported languages (en, zh, hi, es, ar).

## Upgrade notes

No breaking changes. Hard-reload once after upgrading so the browser picks up the
new cache (`remotepower-shell-v5.1.0`).
