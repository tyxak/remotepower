# v6.1.1 — "HardenMatters"

A broad correctness-and-coverage pass across the whole product: real
cross-tenant security fixes, step-up re-auth, litigation hold, a structured
threat model, a full accessibility pass, real per-package patch-pin
enforcement, invoice PDFs and payment-webhook reconciliation, a fleet query
engine, distributed tracing, and a long tail of smaller features and fixes
— several of them found live during production upgrades of v6.1.0. No
breaking API changes.

## Security & tenancy

- **Step-up re-auth for privilege escalation**: `POST /api/auth/step-up`
  re-verifies the caller's own password or TOTP and stamps a short-lived
  freshness window on the session. Wired into the two clearest
  privilege-escalation targets — creating a new admin, and promoting an
  existing user to admin — so a hijacked session token alone can no longer
  silently mint a backdoor admin.
- **Fixed a real cross-tenant bypass**: an API key with no tenant of its own
  defaulted to a display field that resolved to `DEFAULT_TENANT`, meaning any
  admin-role API key became a full cross-tenant superadmin key once tenancy
  enforcement was on. Keys now get their own `tenant_id`, stamped at
  creation; `handle_apikeys_list` is properly tenant-scoped too.
- **Fixed a second real cross-tenant bypass**: the instance-wide SSO
  group-role mapping was gated only on `require_admin_auth()`, so any
  tenant's own admin could silently overwrite role mapping for every other
  tenant's users. Now gated on `_caller_is_superadmin()`.
- **Fixed the API-key tenant fix being reopened one hop removed**: creating
  or rotating a key, and `GET /api/me`, resolved the caller's tenant via the
  older `_user_tenant()` lookup instead of `_caller_effective_tenant()` — so
  an API-key caller (not a session user) minting/rotating a key, or reading
  its own `/me`, could hit the same free-text-field fallback the original
  bypass exploited. All three now use `_caller_effective_tenant()`.
- **Fixed a real cross-tenant leak in saved query templates**: the read/
  delete handlers checked a bare admin-role flag and an unscoped `'shared'`
  visibility flag, neither aware of tenancy — any tenant's own admin could
  see (or fully enumerate) every other tenant's saved query filters, which
  can encode a specific site, customer, or CVE. Templates now stamp their
  owner's tenant at creation and are gated the same way the device roster
  already is; a cross-tenant delete 404s instead of leaking the id's
  existence.
- **Fixed litigation hold not covering metric-rollup pruning**: the hourly/
  daily metric-rollup cadence sweep was an independent aging path that never
  checked the hold, so metric-sample history kept aging out on schedule even
  while a hold was active. Now gated the same way `_purge_old_data()` is.
- **Fixed a job-queue lease-reclaim correctness gap**: a job reclaimed after
  exceeding its running lease could have its outcome silently clobbered by
  the original, still-alive (not crashed) attempt finishing late. Claims now
  carry a token; a stale attempt's completion/failure call is a no-op once
  its job has been reclaimed by a newer attempt.
- **Litigation hold** (Settings → Maintenance): suspends `_purge_old_data()`
  (the one function that age-based-deletes command history, fleet events,
  webhook logs, monitor history, resolved alerts, and metric samples)
  fleet-wide. A reason is required to enable; both directions are audited.
- A structured **STRIDE threat model** (`docs/threat-model.md`), organized by
  attacker goal rather than by feature, complementing the existing narrative
  `docs/security.md`.
- **Guided API-key rotation**: `POST /api/apikeys/{kid}/rotate` mints a
  replacement key carrying the same name/role/scope/rate-limit/IP-allowlist,
  reveals the new secret once, and deactivates the old one.
- **Versioned audit-HMAC key rotation**: `POST /api/audit/rotate-hmac-key`
  rotates the hash-chain signing key without invalidating any prior entry's
  verifiability — each entry records which key generation signed it, so a
  chain spanning a rotation still verifies end-to-end.
- Storage actions (ZFS/Btrfs scrub/balance/destroy/delete) and Proxmox
  snapshot rollback now go through the same 4-eyes approval hook other
  destructive actions use, and Proxmox rollback's typed confirmation is
  checked server-side, not just in the UI.

## Patch management

- A new **Package Snapshots** page: freeze the fleet's current
  installed-package state into a named, immutable snapshot; diff two
  snapshots (added/removed/changed); promote a snapshot to a device tag as a
  pin reference.
- A promoted (pinned) tag now actually **pauses auto-patch dispatch** for
  every device it covers, across every target type — clearing the promotion
  restores dispatch. Drift reporting keeps running regardless, so drift stays
  visible while dispatch is held back.
- **Real per-package pinned-install enforcement**:
  `POST /api/patch-snapshots/{id}/enforce` diffs a snapshot's pinned
  versions against each covered device's installed packages and queues an
  exact apt/dnf/yum install-or-downgrade command over the agent's existing
  generic `exec:` channel — self-detecting the package manager at agent
  runtime. `pacman` is refused with a clear error (no version-pinned install
  against sync repos exists there) rather than silently attempted. Goes
  through the same 4-eyes approval gate as any other `exec:` command.

## Billing & VPN

- **Real generated invoice PDFs**: `GET /api/invoices/{id}?format=pdf`,
  tamper-evidenced the same way the audit-archive download is.
- **Two-way payment-webhook reconciliation**: `POST
  /api/billing/payment-webhook` is a provider-agnostic, shared-secret
  authenticated sink any processor's own webhook (or a thin relay) can post
  into — idempotent on `external_ref`, records payments/refunds, and derives
  a new `partially_paid` invoice status.
- VPN clients get an optional **preshared key**, AES-GCM encrypted at rest
  and generated by default; per-peer RX/TX history with a sparkline; and a
  saveable default tunnel template pre-filled on create.

## Discovery, query & observability

- **Scheduled LAN netscan**: the one-shot netscan is now a living,
  auto-refreshed unmanaged-host feed; discovered hosts fold into the
  topology graph as dashed, muted nodes/edges. OPNsense DHCP-lease reading
  reaches parity with RouterOS via the Kea plugin.
- A new **Data Explorer** page: a whitelisted predicate-tree query engine
  over devices/CVEs/drift (deliberately not raw SQL), plus a batch endpoint
  for up to 10 queries in one call.
- **Real OTLP distributed tracing** — one span per HTTP request, recorded
  from the single request choke point, opt-in and separate from the
  existing OTLP metrics push.
- Structured, correlation-ID-carrying logging for five more request-scoped
  failure paths; conditional-GET/ETag support on the timeline and
  risk-overview reads; and alert-storm **incident auto-promotion** — the
  same event firing on several distinct devices at once now auto-opens a
  status-page incident.

## Accessibility & i18n

- A full **WCAG AA** color-contrast pass (accent text, muted text, avatar
  chips, across all thirteen themes) and the last remaining
  nested-interactive structural fix (the sidebar favorite-star). The
  axe-core accessibility gate (`tests/test_a11y_axe.py`) is now fully
  enforced with zero exemptions.
- German (`de`) translation coverage completed — the last language gap in
  the shared translation dictionaries.

## Also in this release

- A generated Python SDK (`make sdk`, via `openapi-python-client`) — and a
  real invalid-`$ref` bug in the OpenAPI spec found and fixed along the way,
  which would have broken any strict OpenAPI-consuming tool.
- Guided storage provisioning (RAID/LVM/mkfs, whole-disk only, five
  recipes), with every interpolated parameter validated server-side against
  a strict allowlist before it reaches a shell command.
- A UPS-critical threshold alert with opt-in auto-shutdown of dependent
  devices — off by default on two axes (a global toggle and a per-device
  mapping).
- A connector-plugin repository panel (Settings) with metadata and a
  reload-without-restart action.
- Folder-as-tar streaming downloads in the file manager, for directories too
  large for the existing base64-blob channel.
- Per-ticket-type SLA overrides and type-based auto-routing.
- A real total-count pagination envelope on `GET /api/devices`
  (`?meta=1` → `{items,total,limit,offset,next}`), fully backward-compatible.
- An online catch-up pass for the Postgres migration path, mirroring the
  one the JSON↔SQLite path has had since v3.12.0.
- Fixed a real, silently-broken performance regression: the command queue
  (`commands.json`) has been misclassified as "cold" storage since v5.0.0
  due to a filename typo in the entity-promotion migration, falling through
  to a whole-fleet-blob read on every single heartbeat.

## Found live during v6.1.0 production upgrades

- Postgres port/password drift on multi-instance boxes: the install/setup
  scripts now auto-detect the real listening port everywhere (not just some
  call sites), pin every peer-auth call to it, and reuse a marker's already-
  recorded password on a re-run instead of minting a new one each time.
- A service-worker navigation-preload race that intermittently broke SSO
  login in Chromium-based browsers — two physical requests went out for one
  OIDC callback, and the single-use server-side state was consumed by
  whichever one the server saw first.
- The demo site was running its full ~33-sweep maintenance cadence in-line
  on every request instead of out-of-band, and its generated nginx vhost
  swallowed `/sw.js`/`/manifest.json` with a 404.
- An unconditional `listen [::]:80` broke `nginx -t` outright on hosts with
  IPv6 disabled at the kernel level.
- Scanner containers were missing the raw-socket capability `nmap` needs for
  `-sV` and its default NSE script set — scans ran but silently produced
  degraded results.
