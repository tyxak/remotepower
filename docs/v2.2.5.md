# v2.2.5 release notes

Released: 2026-05-15.

Five UX fixes from live driving of the 2.2.4 dashboard.

## Changed

### 1. Container width: 1100 → 1300 px

The dashboard's data density grew through the 2.2 cycle — minimal
devices table, Home tile grid, drift fleet roster — but the page
container stayed at 1100 px from much earlier days. 1300 px fits
four Home tiles comfortably and gives breathing room to wide tables
on standard 1920-monitor setups. Sub-1080 px viewports are
unchanged — the mobile breakpoint takes over below 720 px.

### 2. Scroll wrap when tables / grids exceed 20 rows

Long device lists and busy CVE / patch / monitor tables used to
push the page to several thousand pixels — you'd lose the Home
tile row entirely off the top of the viewport. v2.2.5 wraps any
table tagged via `tableCtl.register()` in a fixed-height scroll
container with a **sticky thead** when the rendered row count is
above 20. Same threshold for the devices card-grid view.

Implementation: new `.scrollable-table-wrap` and
`.scrollable-grid-wrap` CSS classes. `tableCtl.render()` toggles the
table-wrapper class on every render based on row count, so
filtering down below 20 rows removes the wrap, and expanding back
restores it. The thead becomes `position: sticky` so column
headers stay pinned while you scroll.

Custom dark-theme scrollbar tuned to be findable but not intrusive.

### 3. Home → Recent activity items are clickable

Each activity row now routes to the most relevant page or modal
for that event class:

- `device_offline` / `device_online` → device detail modal (if the
  device id was in the payload) else Devices page
- `drift_detected` → Drift page + auto-opens the per-device drift
  detail modal
- `cve_found` → CVE page
- `patch_alert` → Patches page
- `monitor_down` / `monitor_up` / `metric_*` → Monitor page
- `service_down` / `service_up` → Services page
- `container_stopped` / `container_restarting` / `containers_stale`
  → Containers page
- `log_alert` → Logs page
- `command_queued` / `command_executed` → History page

Cursor changes to pointer on hover; tooltip says "Click for
details". The routing helper `_homeActivityAction()` is a switch
statement with a case for each canonical fleet event — a contract
test asserts every entry in the server's `WEBHOOK_EVENT_NAMES`
tuple has an explicit case (no event silently lacks routing).

### 4. Favicon stays at document root

Live testing surfaced an actual deploy gap: `server/html/favicon.png`
existed in source but `deploy-server.sh` only published `*.html`
files from the document root, so `/favicon.png` returned 404 in
the browser. v2.2.5 adds an explicit loop in the deploy script
that publishes `favicon.*`, `robots.txt`, and `manifest.json` from
`server/html/` to `/var/www/remotepower/`.

A duplicate `static/img/favicon.png` was removed — single source
of truth at the root. The `<link rel="icon">` tag still resolves
to `/favicon.png` from any page.

### 5. Detail / Logs / Run hover affordance removed

The hover-revealed action strip was persistently fiddly:

- v2.2.1: introduced it; focus ring clipped on the narrow last cell.
- v2.2.2: moved into the first cell to fix the clipping. Still
  felt clunky in placement.
- v2.2.5: **removed entirely.**

The row dropdown chevron in the actions cell already exposes the
same commands (Detail, Run, Logs, Restart, Shutdown, etc.) and is
keyboard-friendly. Clicking the device name in the row opens the
detail modal — that covers the most common action with no
hover-only affordance.

CSS classes `tr.has-hover-actions` and `.row-actions` are kept as
no-ops (`display: none`) so any HTML still tagged with the class
doesn't break. The `openLogsForDevice` helper is preserved in case
something else wants to reuse it.

## Tests

`tests/test_v225.py` (14 tests):

- `TestContainerWidth` (1): `.container max-width: 1300px`, old
  1100 px gone.
- `TestScrollWrap` (5): `.scrollable-table-wrap` and
  `.scrollable-grid-wrap` CSS present; sticky thead;
  `_applyScrollWrap` helper exists; `SCROLL_THRESHOLD = 20`;
  devices minimal and devices card-grid apply the class.
- `TestActivityClickable` (3): each rendered item has onclick;
  routing helper covers every fleet event in the server's
  `WEBHOOK_EVENT_NAMES`; `cursor: pointer` set.
- `TestFavicon` (3): root favicon exists; no duplicate under
  `static/img/`; deploy script publishes it.
- `TestHoverActionsRemoved` (2): no `class="row-actions"` in row
  template; CSS rule is a `display: none` no-op.

Pre-existing `test_v222.TestPolishHotfixes` updated — three tests
that asserted the *presence* of the hover affordance are now
documented historical records asserting its *absence*. The
evolution chain (2.2.1 introduced → 2.2.2 fixed clipping →
2.2.5 removed entirely) lives in the test comments.

**Total: 865 tests, all passing.**

## Upgrading from 2.2.4

Drop-in. The favicon deploy fix only applies on the next run of
`deploy-server.sh` — if `/favicon.png` was already 404 on your
server, redeploy to publish it.

## Honest caveats

1. **Browser still untested.** All 865 unit tests pass and the
   CSS / JS / Python all parse, but I haven't loaded this in a
   real browser. Most likely-to-need-attention spots: sticky thead
   behaviour when the table is inside a `.table-card` with its
   own `overflow: hidden` (might need to relax the wrap CSS); the
   `.scrollable-grid-wrap` class being added to `#devices-container`
   which already has the `devices-grid` class — both should
   coexist cleanly but worth eyeballing.

2. **Threshold is hardcoded at 20.** No user setting; same number
   for all tables. Reasonable default but if you regularly want
   to see, say, all 50 of your patch-alert devices on one screen,
   currently you can't. Could be made configurable via
   `ui_prefs.scroll_threshold` in a future release.

3. **Activity items always route somewhere.** The default branch
   of `_homeActivityAction()` falls back to opening the device
   detail if a device id is present — otherwise the onclick is
   empty and clicking does nothing. The contract test asserts
   every known fleet event has an explicit case, so this default
   only ever fires for malformed entries or future event types
   not yet in the switch.

4. **The deploy-script favicon fix won't auto-clean a stale
   `/var/www/remotepower/favicon.png`** from a previous
   misconfigured upload — but the install will be a no-op overwrite
   if the file is already there. Worst case: re-run `deploy-server.sh`.

5. **Hover-affordance removal also dropped the keyboard "Logs"
   shortcut.** Previously a keyboard user could tab into the row
   and hit the Logs button. With the strip gone, Logs is reachable
   via the chevron dropdown only. Acceptable tradeoff — the
   strip wasn't keyboard-discoverable anyway (you had to know it
   appeared on hover) — but if power-keyboard-users complain,
   adding a `/` keyboard shortcut for the focused row would be a
   small follow-up.
