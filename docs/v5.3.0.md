# RemotePower v5.3.0 — "ResolveMatters"

A built-in **ticket system** turns the alerts you see into work you can own,
track and resolve — without leaving RemotePower or paying for a separate
helpdesk. Plus an internal **Contacts** directory, and a project-wide hardening,
performance and consistency sweep. No breaking changes.

After upgrading, hard-reload the dashboard once so the new front-end loads
(service-worker cache `remotepower-shell-v5.3.0`).

## Built-in ticket system (opt-in)

Turn it on under **Settings → Advanced → Tickets** (off by default). See the full
[ticket-system guide](ticket-system.md).

- **Tickets.** Stable `#RP000042` numbers; types **Incident / Request / Change**;
  priorities **P1 Major … P4 Low**; statuses Ongoing / Pending customer / Pending
  internal / Resolved / Closed. Open one from an alert (inherits its severity,
  device and title) or stand-alone.
- **Ownership & teams.** Take ownership (assigns you + your team), or hand a ticket
  to a group's queue. Set your team under **Profile → Team**. The Tickets page
  splits into **New / My open / My team's open / Other**, all searchable + sortable.
- **SLA targets.** Per-priority response targets (Settings → Advanced); tickets show
  the **time left**, and breaches raise a `ticket_sla_breached` event so they page
  through your normal alert/webhook routing instead of going unnoticed.
- **Master & sub-tickets.** Link a ticket to a parent; the list nests children and
  closing a master offers to close them too.
- **Alert ↔ ticket loop.** Opening a ticket from an alert acknowledges it; resolving
  or closing the ticket **resolves the linked alert** automatically.
- **Email.** Send via your existing SMTP (with a per-user **HTML signature**) and
  receive replies through a dedicated IMAP mailbox — threaded onto the right ticket
  by `#RP` number, or auto-creating a new ticket, with a mail-loop guard.
- **AI grounding.** A new **Helpdesk triage** advisor and a `tickets` RAG source let
  the assistant answer "what's open for host X?" / "what's breaching SLA?".

## Internal contact directory

A shared team phonebook under **Admin → Contacts** (name / role / company / email /
phone / notes) — searchable, sortable, click-to-mail/dial. See [contacts.md](contacts.md).

## Hardening, performance & polish

- **Security.** Whole-project review + pentest: bandit / gitleaks / semgrep / CodeQL
  all clean, strong live headers (full CSP, HSTS preload, COOP/CORP). Fixed a
  strict-mode alert-permission edge case via the ticket path and added email-header
  injection hardening. See [security-review-5.3.0.md](security-review-5.3.0.md).
- **Performance.** Removed redundant per-heartbeat config/device reads on the hot path.
- **Consistency.** Box-overflow caps, Hindi localisation fixes, and SLA / assignee
  UX refinements.
