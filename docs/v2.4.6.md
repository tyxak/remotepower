# v2.4.6 release notes

Released: 2026-05-18.

A documentation release: a full audit of the feature docs, a new
install &amp; admin guide, and a more useful update-available notice.

## Documentation audit

After ~20 releases the written feature inventory had drifted from
what actually shipped. `docs/features.md` had zero mention of
Proxmox, configuration drift, the mailbox monitor, the MCP server,
or several other 2.2-2.4 features.

`docs/features.md` now has an "Added in 2.2.x - 2.4.x" section
covering: configuration drift detection, the MCP server, the fleet
event log, Proxmox virtualization, LXC and snapshots, the CVE
severity-accuracy work, the default SSH username and quick SSH
link, the mailbox monitor, and the on-demand package scan — plus
the newer offline webhook events.

The in-app Documentation page gained one missing card —
**"Scan packages now"** — bringing it level with the shipped
feature set.

## Added

### Install &amp; admin guide

A new `docs/admin-guide.md`: a complete operator's document
covering what the three components are, installing the server and
agents, post-install hardening (password, 2FA, TLS, the command
allow-list, env-var secrets), day-to-day operation, backup and
restore, the upgrade and rollback procedure, and a troubleshooting
table. The short `docs/install.md` remains the quick-start; the
admin guide is the longer companion.

## Changed

### Update-available notice now shows the update commands

RemotePower already checked GitHub for newer releases and showed a
banner — but the banner only linked to the release page. It now
has a **"How to update"** button that reveals the actual commands
(`git pull && sudo bash install-server.sh`), with an explicit note
that RemotePower does **not** update itself: upgrading is a
deliberate manual step.

This is a deliberate design choice. A self-applying update button
would be a self-modifying remote-code-execution path on the fleet
controller, and could not safely handle backups, agent/server
version skew, migrations, or rollback. Detection and clear
instructions — yes; self-modification — no. See
`docs/admin-guide.md` §7.

## Upgrading from 2.4.5

Drop-in. This release is documentation and one frontend
enhancement — no API, agent, or data changes.

## Honest caveats

1. **No self-updating button — by design.** Covered above. If you
   want a faster upgrade path, the admin guide's procedure is the
   supported one.

2. **Browser-untested**, as usual — the enhanced banner is covered
   by asset tests, not a real browser load.

3. **The first-run setup wizard is not in this release.** A
   guided first-visit install flow was discussed; it is deferred —
   it is stateful UI that overlaps the existing default-password
   banner and cannot be browser-tested here. The written admin
   guide covers the same ground for now.
